🌟 Deteksi Outlier dengan K-Nearest Neighbors (KNN)#

🔍 Apa itu Deteksi Outlier?#

Deteksi outlier adalah proses mengidentifikasi titik data yang berbeda secara signifikan dari mayoritas data lainnya. Outlier bisa muncul karena:

  • Kesalahan pengukuran ❌

  • Variasi alami 🌿

  • Fenomena khusus ⭐

Mengapa penting?

  • Meningkatkan akurasi model 🤖

  • Mengungkap pola tersembunyi 📈

  • Mendeteksi anomali atau fraud 🚨

🤔 Mengapa Menggunakan K-Nearest Neighbors (KNN)?#

KNN adalah algoritma non-parametrik yang menentukan outlier berdasarkan jarak ke tetangga terdekat.

Cara kerjanya:

  1. Tentukan jumlah tetangga k.

  2. Hitung jarak dari tiap titik data ke k tetangga terdekat.

  3. Jika jarak terlalu jauh (di atas threshold tertentu), maka data tersebut dianggap outlier 🚩.

🌟 Euclidean Distance pada KNN#

📏 Rumus Euclidean Distance#

Euclidean Distance digunakan dalam algoritma K-Nearest Neighbors (KNN) untuk mengukur jarak antara dua titik data dalam ruang fitur. Rumusnya adalah:

\[ d(p, q) = \sqrt{(q_1 - p_1)^2 + (q_2 - p_2)^2 + \dots + (q_n - p_n)^2} \]

Keterangan:

  • 📍 \(p\): titik data pertama \((p_1, p_2, \dots, p_n)\)

  • 🎯 \(q\): titik data kedua \((q_1, q_2, \dots, q_n)\)

  • 📊 \(n\): jumlah fitur (dimensi)

✨ Perhitungan Euclidean Distance#

  1. Jarak ke Apel (12, 3.5): 🍭

\[ d = \sqrt{(10 - 12)^2 + (3.0 - 3.5)^2} \]
\[ = \sqrt{(-2)^2 + (-0.5)^2} \]
\[ = \sqrt{4 + 0.25} \]
\[ = \sqrt{4.25} \approx 2.06 \]

  1. Jarak ke Lemon (6, 2.3): 🍋

\[ d = \sqrt{(10 - 6)^2 + (3.0 - 2.3)^2} \]
\[ = \sqrt{(4)^2 + (0.7)^2} \]
\[ = \sqrt{16 + 0.49} \]
\[ = \sqrt{16.49} \approx 4.06 \]

  1. Jarak ke Mangga (14, 4.0): 🍭

\[ d = \sqrt{(10 - 14)^2 + (3.0 - 4.0)^2} \]
\[ = \sqrt{(-4)^2 + (-1.0)^2} \]
\[ = \sqrt{16 + 1.0} \]
\[ = \sqrt{17} \approx 4.12 \]

  1. Jarak ke Jeruk (7, 2.8): 🍋

\[ d = \sqrt{(10 - 7)^2 + (3.0 - 2.8)^2} \]
\[ = \sqrt{(3)^2 + (0.2)^2} \]
\[ = \sqrt{9 + 0.04} \]
\[ = \sqrt{9.04} \approx 3.01 \]

✅ Kesimpulan#

Karena kita menggunakan K = 3 (3 tetangga terdekat), kita mengambil 3 jarak terkecil:

  • Apel 🍭 (2.06)

  • Jeruk 🍋 (3.01)

  • Lemon 🍋 (4.06)

Dari 3 tetangga terdekat, mayoritas adalah Asam 🍋.

Jadi, data baru kemungkinan besar termasuk dalam kelas Asam! 🎉

Semakin kecil jarak Euclidean, semakin mirip data tersebut. KNN menggunakan informasi ini untuk mengklasifikasikan data baru berdasarkan tetangga terdekatnya. 🚀

📚 Kelebihan dan Kekurangan KNN#

Kelebihan:

  • Mudah diimplementasikan dan dipahami.

  • Cocok untuk data dengan hubungan non-linear.

  • Tidak memerlukan proses pelatihan yang rumit.

Kekurangan:

  • Sensitif terhadap data yang memiliki noise.

  • Performa menurun pada dataset yang besar.

  • Memerlukan perhitungan jarak untuk setiap pasangan data.

📐 Penjelasan Perhitungan Euclidean Distance pada KNN pada code#

✨ Rumus Euclidean Distance#

Pada kode ini, perhitungan jarak antara dua titik data \(p\) dan \(q\) dalam ruang fitur menggunakan rumus Euclidean Distance:

\[ d(p, q) = \sqrt{(p_1 - q_1)^2 + (p_2 - q_2)^2 + \dots + (p_n - q_n)^2} \]

Di dalam kode Python, rumus ini diterapkan untuk menghitung jarak antara satu baris data dengan semua baris data lainnya:

\[ d(p, q) = \sqrt{(p_{\text{petal length}} - q_{\text{petal length}})^2 + (p_{\text{petal width}} - q_{\text{petal width}})^2 + (p_{\text{sepal _length}} - q_{\text{sepal_length}})^2 + (p_{\text{sepal_width}} - q_{\text{sepal_width}})^2} \]

  • \(p\) dan \(q\): dua baris data berbeda (titik data)

  • Fitur:

    • petal length (panjang kelopak bunga)

    • petal width (lebar kelopak bunga)

    • sepal_length (panjang sepal/kelopak luar)

    • sepal_width (lebar sepal/kelopak luar)

📏 Rata-rata Jarak K Tetangga Terdekat#

Setelah menghitung jarak antara satu titik data dengan semua titik lainnya, langkah selanjutnya adalah mencari rata-rata jarak dari K tetangga terdekat:

\[ \text{avg_distance} = \frac{d_1 + d_2 + \dots + d_k}{k} \]

dengan:

  • \(k\): jumlah tetangga yang diinginkan (dalam kode: k = 3)

  • \(d_1, d_2, \dots, d_k\): \(K\) jarak terkecil setelah diurutkan

Di kode, ini dilakukan dengan:

avg_distance = sum(jarak[:k]) / k

🚨 Menampilkan Data yang Diduga Outlier#

Setelah kita menghitung jarak rata-rata ke K tetangga terdekat untuk setiap titik data, langkah berikutnya adalah mengidentifikasi dan menampilkan outlier.

💡 Apa yang dilakukan pada tahap ini?

  • Filter data: Kita menyaring titik data yang memiliki jarak rata-rata lebih besar dari threshold tertentu.

  • Menampilkan hasil: Data yang melebihi threshold dianggap sebagai outlier dan akan ditampilkan dalam bentuk tabel agar mudah dianalisis.

📊 Tujuannya:

  • Memastikan kita bisa dengan cepat mengamati titik data yang berpotensi bermasalah.

  • Memberikan gambaran visual terkait seberapa jauh outlier tersebut dari kelompok data utama.

🎯 Kesimpulan Deteksi Outlier dengan KNN#

Berdasarkan implementasi dan visualisasi yang telah dilakukan, kita bisa menarik beberapa kesimpulan penting terkait deteksi outlier menggunakan K-Nearest Neighbors (KNN):

  1. KNN sebagai Metode Outlier Detection

    • KNN efektif untuk mendeteksi outlier dengan mengukur jarak rata-rata ke tetangga terdekat.

    • Titik data yang memiliki jarak lebih jauh dari mayoritas titik lainnya akan diidentifikasi sebagai outlier.

  2. Signifikansi Threshold

    • Penentuan threshold (contohnya, kuantil tertentu seperti 99.99%) penting untuk membedakan data normal dan data outlier.

    • Semakin tinggi threshold, semakin ketat kriteria untuk mendeteksi outlier.

  3. Visualisasi Membantu Identifikasi

    • Plot visual seperti scatter plot membantu memahami pola data dan mengamati titik-titik yang menyimpang.

    • Outlier biasanya terlihat terisolasi dan jauh dari kluster data utama.

  4. Keterbatasan KNN dalam Outlier Detection

    • KNN cenderung kurang efisien jika dataset terlalu besar karena proses perhitungan jarak yang memakan waktu.

    • Sensitif terhadap nilai K — K yang terlalu kecil bisa membuat banyak false positive, sementara K yang terlalu besar bisa menyamarkan outlier.

🌟 Kesimpulan Utama: KNN menawarkan pendekatan intuitif untuk mendeteksi outlier, namun hasil terbaik diperoleh dengan kombinasi pemilihan K yang tepat, threshold yang sesuai, dan visualisasi yang informatif.
